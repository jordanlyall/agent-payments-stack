## Design Context

### Users
Split between technical builders (engineers/founders evaluating payment infra) and market observers (VCs, analysts, researchers). Both came in through organic sharing from a 24K-view tweet. They're crypto-adjacent but not crypto-maximalist — they want signal, not hype. Context: typically viewing on desktop, sharing screenshots.

### Brand Personality
Authoritative, sharp, editorial. The site is a research artifact, not a product page. It has earned credibility by being thorough and correct, not by being loud. Like a well-produced infrastructure report that someone screenshots and drops in a Slack thread.

### Aesthetic Direction
"Research intelligence" — Electric Capital developer report meets Stripe documentation meets a well-edited fintech newsletter. Dense, data-forward, and confident. NOT: crypto neon glows, VC landing page gradients, SaaS rounded cards, AI startup cyan-on-dark. Dark mode is right for this audience and use case — but tinted warm, not pure black.

### Design Principles
1. **Earned authority over claimed authority** — every visual element should signal rigorous research, not marketing. Data wins over decoration.
2. **Editorial density is a feature** — this is meant to be comprehensive. Don't fight the density; make it legible and hierarchical.
3. **Warm neutrals, not cool tech** — the palette should feel like premium print material, not a dark-mode developer tool.
4. **Taxonomy clarity** — the chip/tag system is the core UX. Status distinctions (live, early, announced, acquired) must be immediately readable.
5. **No crypto kitsch** — no neon, no glows, no token-price energy. This is infrastructure research, not a launch page.

### Technical Notes
- Static HTML, no framework
- Fonts via Google Fonts (currently Inter + JetBrains Mono)
- Vercel deployment
- Must preserve all interactive functionality (search, filter, subscribe form, tooltips)
