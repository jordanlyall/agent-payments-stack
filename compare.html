When to Pick Each
x402
The default choice for EVM builders
Pick this if
  • You're building on Base, Ethereum, or any EVM chain
  • You want HTTP-native payments with no extra infrastructure
  • You're using ElizaOS, Coinbase AgentKit, or Vercel/Cloudflare/AWS
  • You want open-source you can audit and fork
  • You need Coinbase ecosystem compatibility
Skip if you need Bitcoin-native payments or are deep in the OpenAI/Stripe stack.
Tempo MPP
For complex service networks and enterprise
Pick this if
  • You're building multi-party service coordination (not just pay-per-call)
  • You're in the Anthropic or enterprise AI ecosystem
  • You need 100+ services on a shared coordination layer
  • You prefer managed infrastructure over self-hosted
Skip if you want on-chain settlement or open-source. Opaque by design.
L402 / Lightning
For Bitcoin-native, sub-cent micropayments
Pick this if
  • Your users or stack is Bitcoin-native
  • You need true sub-cent micropayments (fractions of a sat)
  • You're already running Lightning infrastructure
  • Latency and finality on Lightning rails are acceptable
Skip if you need EVM compatibility, USDC settlement, or don't want to run Lightning nodes.
ACP
For the OpenAI / Stripe developer stack
Pick this if
  • You're building with OpenAI APIs as your agent runtime
  • You're already using Stripe for payments
  • You want the most familiar auth and payment UX for end users
  • You need fiat-friendly settlement alongside crypto
Skip if you need pure on-chain settlement or EVM-native agent infrastructure.
Editorial Take · March 2026
x402 is the leading standard. AWS, Cloudflare, and Vercel natively support it. It has the most GitHub stars (5,800), the most ElizaOS integrations, and the most open infrastructure. Coinbase, Anthropic, and Stripe are all building around it. That is not an accident.

The real race is x402 vs ACP. Both launched Nov 2024. Both target HTTP-native agent payments. The difference is ecosystem: x402 is on-chain and EVM-native; ACP is Stripe-native and developer-familiar. A world with both is likely. A developer on Base builds on x402. A developer on OpenAI builds on ACP.

L402 solves a different problem. If you need Bitcoin-native or Lightning rails, it is the only answer. If you don't, it is not competitive with x402 on simplicity or EVM ecosystem support.

Google AP2 is the wildcard. 60+ enterprise partners and Google distribution is not nothing. But "announced" means no spec, no code, no timeline. File it under "watch closely."

Tempo MPP is the enterprise layer, not the open layer. $500M Series A and 100+ services are real signals, but the lack of open-source and on-chain settlement limits developer adoption. It will dominate where managed infrastructure is preferred.
